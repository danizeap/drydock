# Decision Log

## Change

codex-mutate-scope

## Decisions

| Date | Decision | Reason | Alternatives Considered |
| --- | --- | --- | --- |
| 2026-07-24 | **Meter before optimising** — cost reporting ships first, ahead of any scoping | The report gave one number (553,941) from one run on one repo. We had no per-run cost at all, so any "fix" would be judged by vibes and the thresholds for everything else would be invented. The gauge already existed; the most expensive operation in the system was simply running behind it with no meter | Ship `--files` first and assume it helps (rejected: unfalsifiable, and it is the *hypothesis* under test); estimate cost from repo size (rejected: an estimate is what you build once you have measurements, not instead of them) |
| 2026-07-24 | The **fuel delta is authoritative**; token counts are informative | The delta is what the account was actually charged and is what runs out at the end of a week; Codex's `usage` is self-reported and its spelling has changed between versions. Both are reported, but only one is called the cost | Report tokens only (rejected: tokens do not map linearly to the quota that actually binds); report the delta only (rejected: tokens are the cross-check when the gauge lags) |
| 2026-07-24 | Unmeasurable cost is `null`, never `0` | Same principle already settled for the fuel gauge: a fabricated number fails silently. `0` reads as "this run was free" — the Owner budgets against a fiction and the shortfall arrives with no warning at all | Default to 0 (rejected: silently false); omit the keys (rejected: a consumer cannot distinguish "free" from "not reported") |
| 2026-07-24 | A negative delta (window reset mid-run) reports `null` | `after < before` means the window rolled over, not that the run refunded fuel. Reporting the arithmetic would be worse than reporting nothing | Clamp to 0 (rejected: indistinguishable from a free run); report the negative (rejected: nonsense in a field named "used") |
| 2026-07-24 | `--files` is **opt-in**, not the default | Field data: the reporter's usage is bimodal. Small-and-coupled (1–5 files) is the frequent case and scoping is a clear win; wide-and-mechanical (rename a symbol, add a parameter to every call site) genuinely wants whole-repo context and scoping would hurt. A default that is wrong for a whole mode is a worse default than none | Scope by default (rejected: breaks the sweep case); infer the mode from the task text (rejected: guessing intent from prose, unfalsifiable and surprising) |
| 2026-07-24 | Scoping is **SOFT** — inline the targets, disclose what falls outside, block nothing | The reporter's own 3-file change was a tool body, a test's expected-tools list, and i18n labels: coupled files not discoverable from the target's name. The files an operator forgets to name are exactly the ones that complete the change, so a hard boundary converts an under-scoped run into a red gate caused by the scoping rather than the code — the N/A-vs-FAIL trap in new clothes. Disclosure also *improves* review by naming where to look first | Hard scope via sparse checkout (rejected: tests need a runnable tree, so it breaks the gate; held in reserve pending measurement); hint-only with no inlining (rejected: leaves Codex crawling for the files, which is the cost being attacked) |
| 2026-07-24 | Inlined targets get `review.py`'s full secret treatment, refused **before** spawning | Inlining sends file content off-machine — the identical exposure `review.py` guards, reached by a different door. An operator naming `.env` as an edit target is asking for something forbidden, so the answer is refusal, and it must land before a process starts rather than after | Guard by name only (rejected: an ordinary-looking file holding a key is exactly what auto-inlining surfaces); skip-and-continue (rejected: explicit naming deserves a refusal, matching the review-path posture) |
| 2026-07-24 | Diff shape compares **structural signatures**, not raw text | "Add a parameter to every call site" adds a different literal line to each file, so raw-text similarity scores a mechanical sweep as divergent and the advisory would cry wolf on the exact case it is meant to stay quiet for. Erasing identifiers and numbers while keeping punctuation captures *same structure, different names* — which is what a sweep is | Raw-line Jaccard (rejected: sweeps score as divergent); sequence similarity on whole diffs (rejected: order-sensitive across files and quadratic on large text); ask a model to judge (rejected: costs tokens to save tokens, and is non-deterministic) |
| 2026-07-24 | The diff-shape signal is an **advisory that never gates** | It encodes a human judgment ("a dozen files of judgment-heavy work is where the gate means least") on provisional thresholds. Encoding a heuristic as a blocker would fail the review-burden case in the *blocking* direction, which invites bypass — the same reasoning that keeps the coverage-gap advisory non-gating | Block wide-divergent diffs (rejected: heuristic thresholds must not stop work); stay silent (rejected: the report identified this as the highest-risk delegation shape and nothing surfaced it) |
| 2026-07-24 | An absent measurement never reports the reassuring case | Three of the four blockers were one mistake wearing different clothes: a boolean gauge yielding `0`, `total_tokens` folding a missing half into zero, and a deletion-only diff reporting `wide-repetitive`. Each turned "we did not measure" into an affirmative claim. R1 was written to forbid exactly this and the same file violated it twice more, which is why `unknown` now exists as a first-class shape | Default to the benign case (rejected: that is the bug, three times); omit the field (rejected: a consumer cannot tell "measured benign" from "not measured") |
| 2026-07-24 | The diff parser is **hunk-aware**, because the reviewed party writes the diff | An added source line reading `++ b/x.py` renders as `+++ b/x.py`, so a prefix-matching parser read it as a file header and stole the rest of that file's lines — Codex suppressing its own advisory. This is the same class as a file closing its own fence; I had fixed it on the prompt side and left it open on the parse side. `@@` counts now bound the body, so content can never be read as structure | Escape or strip suspicious added lines (rejected: mangles the content being measured); trust git's output (rejected: git is faithful, but what it faithfully reproduces is attacker-authored content) |
| 2026-07-24 | A regression test must be proven to **fail** against the defect it guards | The confirmation pass found `test_added_content_cannot_hijack_the_diff_parser` passing against the vulnerable parser: the poison line was the last line of the last file, so the broken parser had nothing left to misattribute. That is the loose-requirement failure one level down — a guard that cannot detect its own regression. Discrimination is now demonstrated against a reimplemented pre-fix parser, not assumed | Trust that a test written after a fix must cover it (rejected: this one did not, and nothing surfaced that); rely on the verifier to re-derive it each time (rejected: verification is not a substitute for a suite that holds on its own) |
| 2026-07-24 | Thresholds are provisional and **emitted with the measurement** | `WIDE_DIFF_FILES=8` / `LOW_REPETITION=0.45` are reasoned, not measured. Shipping the numbers inside the result means a later reader can re-derive the judgment instead of trusting a verdict computed from constants they cannot see | Hard-code silently (rejected: an unexplainable advisory is ignorable noise); wait for calibration data (rejected: the data arrives from using this, so it would never start) |
