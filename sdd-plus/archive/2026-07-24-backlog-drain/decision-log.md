# Decision Log

## Change

backlog-drain

## Decisions

| Date | Decision | Reason | Alternatives Considered |
| --- | --- | --- | --- |
| 2026-07-24 | `--abandon` is a disposition DISTINCT from `--force`, recording the absence of a verification | The 48 backlog packets are abandoned mid-lifecycle; `--force` exists to waive a specific gate on work you *stand behind*, the opposite intent. Using `--force` to clear them would leave no honest record that they were never verified — and the design panel was explicit that draining must never fabricate a PASS. `--abandon` writes `Abandoned — never verified` and an Override, so the archive tells the truth | Reuse `--force` (rejected: wrong intent, no honest "never verified" record); a `discard` command that deletes (rejected: archive only ever MOVES — never lose work); synthesize a PASS to clear the gate (rejected: the exact fabrication the whole packet forbids) |
| 2026-07-24 | Abandon and force are mutually exclusive | They are opposite stances — force says "I stand behind this, waive the gate"; abandon says "I do not, bury it unverified". Allowing both is incoherent and muddies the audit record | Silently prefer one (rejected: hides operator intent); allow both (rejected: contradictory dispositions on one packet) |
| 2026-07-24 | Triage is strictly read-only and robust to a broken packet | The messiest backlog packets are precisely the broken ones (missing files), exactly where a naive classifier would crash and abort the whole sweep — leaving the operator unable to even *see* the backlog. Per-packet try/except → UNKNOWN guarantees the sweep always completes | Let a broken packet raise (rejected: aborts the sweep on the packets that most need triaging); skip broken packets silently (rejected: they vanish from the report — can't act on what you can't see) |
| 2026-07-24 | Abandon records an Override (like a forced archive), so brief.py demotes it | brief.py's promise ladder already demotes forced archives and grants no rung to hand-moved dirs. An abandoned packet is the strongest "not positive evidence"; routing it through the same Override machinery keeps the Owner brief honest with no new brief.py logic | A distinct abandon record brief.py must learn to read (rejected: new coupling for no gain — Override already means "not clean") |
| 2026-07-24 | Abandoned packets go to `archive/` with the honest Result marker, not a distinct dir | Reuses the existing move machinery; the `Abandoned … never verified` Result + the Override make them distinguishable on inspection, and they leave `changes/` so orientation is clean. A distinct location is a deferred Owner decision | Prefix the dir `abandoned-…` (rejected for now: risks confusing brief.py's archive parser); a separate tree (rejected: deferred to the Owner, not needed to drain) |
| 2026-07-24 | Abandon's entomb warning covers non-canonical/unattributable deltas too (verifier Finding 1) | The verifier caught that `_unsynced_requirements` only sees canonical grammar, so abandoning a `### R5 —` or no-`Capability:` delta buried spec knowledge with NO warning — the one permanently-lossy operation going silent exactly where triage and verify are loud (they flag such deltas as NEEDS-SYNC). It never fabricated a PASS, but "warn when it buries spec knowledge" was only half-met. Abandon now also names deltas whose sync it cannot verify | Leave it scenario-scoped to canonical (rejected: abandon is where silence costs the most — the messy packet is the target case); block abandon on non-canonical (rejected: abandon exists precisely to bury messy packets — blocking defeats it) |
| 2026-07-24 | Collision check precedes any mutation; a stray verdict on a malformed heading is scrubbed; whitespace-only reason refused (verifier Findings 2–4) | Finding 2: abandon wrote the Result + Override before the collision check, so a clash left a half-abandoned packet and a re-run duplicated the Override — the check now runs first, so a clash leaves the packet fully intact. Finding 3: `_replace_result_section` matched only `^## Result$`, so a `## Result: PASS` heading kept its stray PASS — the heading is now matched liberally and rewritten clean, so no verdict survives an abandon. Finding 4: `--reason "   "` was accepted; now `.strip()`-checked | Ship the lows as-is (rejected: each is a small honesty/robustness hole, and cheap to close; leaving them contradicts the packet's own "never a PASS, never lose work" thesis) |
| 2026-07-24 | No dedicated `/drydock:triage` slash command in this slice | The `sdd.py triage` subcommand is documented under `/drydock:archive`; a separate slash command adds plugin-manifest + registry surface for marginal discovery gain. Can be added later if the drain flow proves common | Add the slash command now (deferred: keep the slice tight; the subcommand is already reachable and documented) |
