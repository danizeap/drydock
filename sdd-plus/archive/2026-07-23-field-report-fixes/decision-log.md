| 2026-07-23 | An untrustworthy test result becomes a NEW verdict `unverifiable` with `clears: false` ‚Äî never `green` | A false green on a review gate fails in the DANGEROUS direction (it invites the merge the gate exists to guard). | 2026-07-23 | Trust is an ALLOW-LIST: a command is trusted only if it is a SIMPLE command, optionally `&&`-chained. Everything else is untrusted | **Corrected after the verifier returned NOT VERIFIED on the first attempt.** The first pass was a blacklist that detected only `\|` ‚Äî and the verifier falsified it with working counterexamples on both shells, including on the Owner's own platform: `test & echo hi` (cmd.exe sequencing) and `'x\|echo hi'` (cmd.exe does not quote with `'`, so a naive POSIX scanner walks past a real pipeline) both still produced `green` on a FAILING test. A blacklist on an open-ended set of shell constructs cannot be complete; an allow-list makes an unrecognised construct untrusted by default, which is the safe direction. `&&` is the one safe chain (it short-circuits, so failure propagates); `2>&1` is a redirect, not a separator | Blacklist of masking chars (REJECTED after falsification ‚Äî missed `;`, `&`, newline, backtick, `$()`, and cmd.exe quoting); `set -o pipefail` (rejected: not portable ‚Äî `shell=True` is cmd.exe on Windows); parsing test output for failure markers (rejected as primary: framework-specific and brittle) |
