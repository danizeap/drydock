| 2026-07-24 | Split the archive-inversion into Packet A (no living-spec write) and Packet B (the auto-sync writer), and ship A first | A design panel (3 lenses + adversarial critique + synthesis) independently reached the split: a source-of-truth mutation must not ride the same change that makes finishing reflexive, and auto-sync is the one catastrophic-risk piece. Packet A flattens the finish and closes a real bug WITHOUT ever writing a living spec, so it fails safe by construction | Build the whole thing at once (rejected: bundles the risky writer with everything); build the writer first (rejected: highest risk, needs a property test on the real corpus first) |
| 2026-07-24 | One shared `archive_readiness` feeds both the prompt and the archive gate | The failure mode of a ready prompt is telling someone "ready" when archive will block. The only structural defense is that the prompt and the gate read the SAME function — so they cannot drift. Grafted from the panel's safety-first lens | Compute readiness separately in verify and archive (rejected: two predicates drift, exactly the class of bug that produces a false ready) |
| 2026-07-24 | The ready prompt FAILS TOWARD needs-sync — READY only on positive confirmation, never from an empty blocker list | The panel found a pre-existing vacuous-pass hole: `delta_added_requirements` returns [] for non-canonical `### R5 —` grammar, so the "requirement present in living spec?" gate passes vacuously and an unsynced delta reads as ready / archives clean. Inferring READY from "no blockers" would inherit that hole. So READY requires positively confirming synced + canonical | Print READY whenever the blocker list is empty (rejected: that IS the hole — an unverifiable delta produces an empty list) |
| 2026-07-24 | The delta-grammar lint WARNS, it does not REJECT | Every recent packet authored deltas as `### R5 — <name>`; a Python sync script normalized them out-of-band. Hard-rejecting would break the established authoring habit and force `--force` on every archive, including this repo's own. WARN surfaces the issue and makes the prompt honest without breaking the flow; REJECT (which would let archive itself close the hole) is a deliberate Owner decision that also raises Packet B's auto-sync fire-rate | REJECT now (rejected: breaks existing authoring + this repo's own archives); silently normalize (rejected: hides the issue, which is what let it persist); do nothing (rejected: leaves the false-ready hole invisible) |
| 2026-07-24 | The ready prompt lives in `cmd_verify` stdout, not the completion_gate or session_orient hooks | verify's stdout is the exact deterministic moment green is learned, in output the operator is already reading. completion_gate goes SILENT at green (it fires only on verification_pending) — structurally the wrong home. session_orient is session-start timing and is the decayed-noise source being fixed | Emit from completion_gate (rejected: silent exactly at green); from session_orient (rejected: wrong timing, the noise problem) |
| 2026-07-24 | verify's prompt is deliberately STRICTER than archive's gate on delta grammar; archive stays warned-but-permissive | The verifier caught that "prompt and gate can never disagree" was an overclaim — they DO diverge on grammar: verify refuses READY on a non-canonical delta, but archive warns and proceeds (no block, no override), so the "archive clean" half of the vacuous-pass hole is still open. Closing it means making archive treat unverifiable grammar as a (waivable) blocker — a soft-REJECT that forces `--force` or canonical authoring on every archive, including this repo's own non-canonical packets. That is the exact REJECT decision already reserved for the Owner. So 0.12.0 fixes the OVERCLAIM (the docs/test no longer say "archive clean" is closed) and PINS the deferred behavior with a test, rather than silently shipping a framing that says more than the code does | Make archive soft-reject non-canonical now (rejected: unilaterally changes the authoring workflow, an Owner decision); leave the docstring saying "archive clean" is closed (rejected: that is the overclaim — the exact recurring sin — an untested claim the code doesn't back); expand archive_readiness to include grammar (rejected: same soft-REJECT, breaks this repo's own archive flow until canonical migration) |
| 2026-07-24 | Defer the Stop-time archive-ready nudge (the reach-everyone piece) to 0.12.1 | It catches the operator who never runs verify — the actual skipper population — but it needs `archive_readiness` reachable from the hook. That refactor belongs with 0.12.1, not rushed into 0.12.0 | Import sdd.py from the hook (rejected: couples the Stop hook to the CLI); duplicate the predicate in the hook (rejected: two predicates drift — the exact bug we're preventing) |
